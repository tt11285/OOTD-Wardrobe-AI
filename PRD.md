## 执行摘要 Executive Summary

**一句话产品定位：**
> 拍一张衣柜照片，AI 帮你决定今天穿什么。

**产品概述：**

OOTD 是一款 AI Native 的个人衣橱穿搭应用，解决用户「衣橱满满却不知道穿什么」这一高频每日痛点。用户通过照片或视频快速建立数字化衣橱，AI 自动识别每件衣物的品类、颜色、风格；当用户输入今日场合（如"明天面试"、"周末约会"），AI 基于内置的审美知识库（色彩理论 + 风格模板 + 精选博主搭配案例 RAG），从用户真实衣橱中生成 2-3 套个性化搭配方案，并附搭配理由。

与传统衣橱管理 App（如 Stylebook、Whering）依赖手动录入和规则引擎不同，OOTD 完全以多模态 LLM 为核心引擎，识别 + 推荐 + 解释均由 AI 完成，让 AI 不是辅助功能而是产品本身。

**Quick Facts：**

- **目标用户：** 20-35 岁女性，城市白领/大学生，衣物量中等以上但缺乏搭配自信
- **核心痛点：** 每日早晨穿搭决策困难、衣物利用率低、缺乏个人化造型师指导
- **首发平台：** 移动 Web（Mobile-First Responsive）
- **商业模式：** 免费 + 订阅（Freemium）
- **北极星指标：** DAU/MAU ≥ 30%
- **目标节点：** 1 个月内 Beta → 3 个月内 Public Launch

## 1. 问题陈述 Problem Statement

### 1.1 核心问题

> **"早晨 8 点，站在衣柜前，有 50 件衣服，却花 20 分钟选不出来今天穿什么，最终只能穿那件已经穿烂的黑色 T 恤。"**

用户拥有大量服装，却在日常穿搭决策中陷入「选择困难」。这不是衣物数量问题，而是**决策辅助缺失**问题。

### 1.2 用户痛点矩阵

| 痛点层级 | 具体场景 | 发生频率 | 情绪烈度 | AI 干预可行性 |
|---|---|---|---|---|
| **每日决策** | 衣橱满满却不知道穿什么 | 每天 | 中等（烦躁） | ⭐⭐⭐⭐⭐ |
| 购物决策 | 逛街不知道买什么、买了难搭 | 每周 | 中等（纠结） | ⭐⭐⭐ |
| 风格认知 | 不懂自己适合什么颜色/风格 | 长期 | 高（焦虑） | ⭐⭐⭐ |
| 身材适配 | 不知道如何用穿搭扬长避短 | 长期 | 高（自卑） | ⭐⭐ |
| 场合应对 | 面试/约会/重要场合穿搭无指导 | 偶发 | 高（紧张） | ⭐⭐⭐⭐ |

**V1 聚焦：每日决策痛点**（最高频 + AI 价值最直接 + 易形成日常习惯）

### 1.3 现有解决方案及其缺陷

| 现有方案 | 用户实际行为 | 缺陷 |
|---|---|---|
| 刷小红书穿搭博主 | 每天花 15-30 分钟刷帖 | 博主衣服 ≠ 用户衣橱，参考性弱 |
| 问朋友 | 微信发图问意见 | 不够即时，朋友不在线就卡住 |
| 凭直觉随便穿 | 默认穿"安全款"老三件 | 衣物利用率低，浪费投入 |
| 传统衣橱 App（Stylebook 等）| 手动录入分类 | 录入门槛高，无智能推荐 |
| 私人造型师 | 极少数高端用户 | 价格 2000+/月，普通用户用不起 |

### 1.4 用户影响

**个人维度：**
- 每天浪费 15-20 分钟在衣柜前发呆
- 70% 衣物长期闲置（实际穿着率 < 30%）
- 重要场合穿错衣服带来社交焦虑、自我否定
- 重复购买相似单品造成金钱浪费

**市场维度（中国市场）：**
- 中国服装市场规模 3 万亿+，年轻女性人均年服装支出 5000+
- 衣物闲置率 50%+，间接浪费每年数千亿
- 国内"AI + 个人衣橱"赛道几乎空白，竞品停留在手动记录阶段

### 1.5 为什么是现在？

1. **技术拐点已至**：2024-2026 多模态 LLM（Claude Sonnet 4.5 / Gemini 2.5）的视觉识别能力跨过了商业可用阈值，单张图识别 10+ 件衣物准确率 95%+
2. **成本拐点已至**：Prompt Caching + 模型分级使每用户月成本降至 < ¥1，订阅模式可跑通
3. **竞品代际差距**：国际竞品（Whering、Cladwell）仍用传统 CV 模型；国内市场几乎空白
4. **用户行为成熟**：手机相册随手拍 → AI 处理已是 Z 世代默认习惯

## 2. 目标与目的 Goals & Objectives

### 2.1 业务目标（6-12 个月）

1. **用户目标：** 上线 6 个月达到 10,000 MAU，DAU/MAU > 25%
2. **留存目标：** Week 1 留存 > 40%，Month 1 留存 > 20%
3. **商业目标：** 验证 Freemium 订阅模式，付费转化率 > 5%
4. **品牌目标：** 在小红书等渠道建立"中文 AI 穿搭工具第一品牌"心智

### 2.2 用户目标

1. **效率：** 把每日穿搭决策时间从 15-20 分钟压缩到 30 秒
2. **自信：** 在重要场合（面试/约会/会议）穿出得体且有审美的搭配
3. **价值发现：** 让已购衣物的利用率从 30% 提升到 70%+
4. **审美成长：** 通过 AI 的搭配解释，逐步内化色彩 / 风格规律

### 2.3 Non-Goals（明确不做的）

- ❌ 不做电商导流（V1 不引入购买链接，避免变商品推荐）
- ❌ 不做社区社交（不是穿搭分享社区，专注个人决策）
- ❌ 不做完整衣橱管理（不做洗护提醒、过季归档等周边功能）
- ❌ 不做品牌识别（不识别 logo / 品牌，只识别品类风格）
- ❌ 不做虚拟试穿 / AR 换装（技术复杂度高，超出 V1 范围）

## 3. 用户画像 User Personas

### 3.1 主要用户画像：林夕（25 岁）

**Demographics：**
- 年龄：25 岁
- 职业：互联网公司运营专员
- 居住：上海，独居
- 月收入：12,000-15,000 元
- 技术接受度：高（小红书重度用户、用 ChatGPT）

**行为模式：**
- 衣橱有约 80 件衣物，3 个收纳柜
- 每月在淘宝/小红书购物 1-2 次
- 工作日通勤 + 周末逛街/约会 + 偶尔出差
- 早晨 7:30 起床，8:00 必须出门，时间紧张

**需求与动机：**
- 想要看起来"得体且不土"，但不愿花太多时间
- 希望已购衣物物尽其用（毕竟花了钱）
- 想提升自己的审美但没时间系统学习
- 偶尔有面试/相亲/客户会议等关键场合，焦虑感强

**痛点：**
- "衣柜里好多衣服，但每天还是穿那三套"
- "买了不少漂亮单品，但不知道怎么搭进日常"
- "面试前一晚总要焦虑很久不知道穿什么"

**用户原声：**
> _"我不是没有衣服，我是不知道我有什么衣服，更不知道怎么搭它们。每天早上都是一场战役。"_

### 3.2 次要用户画像：周雨桐（22 岁）

**Demographics：**
- 年龄：22 岁
- 职业：大四学生，准备实习/求职
- 居住：北京，宿舍/合租
- 月支出：3,000-4,000 元

**行为模式：**
- 衣物量中等（40-60 件），多为快时尚品牌
- 即将进入职场，对"通勤穿搭"完全没经验
- 重度小红书用户，对"显瘦"、"显高"、"显白"关注度高

**核心需求：**
- 学生气穿搭 → 职业感穿搭的过渡指导
- 面试场合的具体穿搭方案
- 在有限预算下做出最优搭配组合

**用户原声：**
> _"马上要去面试了，但我感觉自己穿得像高中生，不知道怎么穿才像个大人。"_

### 3.3 暂不服务的用户

- 极简主义者（衣物 < 20 件，不需要 AI 辅助）
- 男性用户（V1 优先聚焦女性市场，后续扩展）
- 高端定制用户（私人造型师服务的目标群体）

## 4. 用户故事与需求 User Stories & Requirements

### 4.1 核心 Epic：从「拍照建库」到「AI 推荐穿搭」

#### P0 必做故事（V1 核心）

##### Story 1：快速建立数字化衣橱

```
作为一名有 30-80 件衣服的用户，
我希望通过手机拍照（一次最多 10 张）或上传视频（3-5 件衣服），
让 AI 自动识别每件衣物的品类、颜色、风格，并存入我的衣橱，
这样我就不需要手动录入每件衣服的属性。
```

**验收标准：**
- [ ] 用户可一次性上传 1-10 张照片
- [ ] 单张图片识别准确率 ≥ 90%（品类+主色调）
- [ ] 10 张照片识别总耗时 ≤ 60 秒
- [ ] 识别结果以网格视图展示，可点击编辑修正
- [ ] 衣物属性包含：品类、颜色、风格标签、季节、正式度（1-5）
- [ ] 支持视频抽帧识别（每 0.5s 一帧，去重）
- [ ] 边界场景：图片中无衣物时给出友好提示

**优先级：** P0 Must Have
**工作量：** L
**依赖：** Claude Sonnet 4.5 API、Supabase Storage

---

##### Story 2：自然语言场合输入

```
作为一名时间紧迫的用户，
我希望用一句话或一个标签描述今天的场合（如"明天面试"、"周末约会"），
而不需要填表单选下拉框，
这样我能在 5 秒内完成"输入需求"这一步。
```

**验收标准：**
- [ ] 提供 5 个高频场合快捷标签：通勤、约会、面试、休闲、重要会议
- [ ] 同时支持文字输入框，让用户描述自定义场合
- [ ] AI 能理解场合 + 隐含正式度（如"客户会议" → 正式度 4）
- [ ] 可选：温度/天气信息（手动输入或获取定位）
- [ ] 输入后立即触发推荐生成

**优先级：** P0 Must Have
**工作量：** S

---

##### Story 3：AI 生成个性化搭配方案

```
作为一名想要好看搭配的用户，
我希望 AI 从我真实衣橱里选出 2-3 套搭配方案，
每套方案附带「为什么这样搭」的简短解释，
这样我既能立刻得到答案，又能学到一点搭配知识。
```

**验收标准：**
- [ ] 输出 2-3 套搭配方案
- [ ] 每套包含：上衣 + 下装 + 鞋 + 可选外套/配饰
- [ ] 每套方案的单品都必须真实存在于用户衣橱
- [ ] 每套方案附 50 字以内的搭配理由（说人话，避免空洞）
- [ ] 推荐结果可视化：单品图片拼接展示
- [ ] 生成耗时 ≤ 8 秒
- [ ] 推荐结果引用审美知识库（色彩规则 + 风格模板 + 博主案例）
- [ ] 用户可滑动切换不同方案

**优先级：** P0 Must Have
**工作量：** L
**依赖：** RAG 审美知识库（200+ 案例）、Claude Sonnet 4.5

---

##### Story 4：搭配方案"理由"展开

作为一名想理解搭配逻辑的用户，

## 5. 成功指标 Success Metrics

### 5.1 北极星指标

**DAU/MAU ≥ 30%**

衡量产品是否真正成为用户的日常工具。穿搭决策天然每日发生，如果用户没有每天打开，说明产品价值未被验证。

### 5.2 关键指标（HEART 框架）

| 维度 | 指标 | 目标 |
|---|---|---|
| **Happiness** | App 评分 / NPS | NPS ≥ 40 |
| **Engagement** | DAU/MAU | ≥ 30% |
| **Adoption** | 首次建库完成率（注册→建库 ≥ 5 件） | ≥ 70% |
| **Retention** | Week 1 留存 | ≥ 40% |
| **Retention** | Month 1 留存 | ≥ 20% |
| **Task Success** | 搭配方案接受率（"今天就穿这套"点击）| ≥ 50% |

### 5.3 AARRR 漏斗

| 阶段 | 指标 | 目标 |
|---|---|---|
| **Acquisition** | 小红书种草 → 注册转化率 | ≥ 3% |
| **Activation** | 注册 → 完成首次建库 5 件 | ≥ 60% |
| **Retention** | Week 1 回访次数 | ≥ 3 次 |
| **Revenue** | 免费 → 付费转化率 | ≥ 5% |
| **Referral** | 用户分享搭配卡片 | ≥ 10% |

### 5.4 埋点事件清单

- `user_registered` — 用户注册
- `wardrobe_upload_started` — 用户开始上传衣物图片
- `wardrobe_item_recognized` — 单件衣物识别完成（含成功/失败）
- `wardrobe_item_edited` — 用户修改 AI 识别结果（衡量识别准确率）
- `outfit_request_submitted` — 用户提交场合需求
- `outfit_generated` — AI 完成搭配生成（含耗时）
- `outfit_viewed` — 用户查看搭配方案详情
- `outfit_accepted` — 用户选择"今天就穿这套"
- `outfit_rejected` — 用户切换到下一套（衡量满意度）
- `outfit_shared` — 用户分享搭配卡片
- `paywall_shown` — 触达付费墙
- `subscription_started` — 用户订阅成功

## 6. 范围界定 Scope

### 6.1 In Scope

**V1 必做：**
- ✅ 多图/视频上传 + AI 识别衣物
- ✅ 衣橱网格视图
- ✅ 场合输入（标签 + 自由输入）
- ✅ AI 搭配方案生成（2-3 套 + 解释）
- ✅ 移动 Web 响应式界面
- ✅ 内置审美知识库（200+ 案例）
- ✅ 用户账号系统
- ✅ Freemium 付费墙（建库 2 件 / 推荐 3 次免费）

**V2（V1 上线 2-3 个月后）：**

### 7.5 RAG 检索流程

```
用户输入场合 "明天面试"
  ↓
LLM 提取关键词：{ occasion: "面试", formality: 4, style_hint: "干练" }
  ↓
文本 embedding（text-embedding-3-small）
  ↓
pgvector 余弦相似度检索 → Top 5 案例
  ↓
拼接进 Prompt 的【参考案例】区
  ↓
Claude 推理生成
```

### 7.6 性能优化

| 优化点 | 方法 | 预期收益 |
|---|---|---|
| 图片大小 | 上传前 resize ≤ 1024px | Token 数降 70% |
| 缓存 | System Prompt 启用 Prompt Caching | 单次推理省 90% input cost |
| 批量识别 | 一次调用传多张图（最多 10）| API 调用次数降 10x |
| 衣橱缓存 | 用户衣橱 JSON 客户端 + 服务端缓存 | 减少 DB 查询 |
| Streaming | 搭配生成用 SSE 流式返回 | 首字延迟 < 1s |

### 7.7 安全与隐私

- **图片隐私：** 用户上传的衣物照片仅存储在用户私有空间（Supabase RLS）
- **API Key 安全：** Anthropic API Key 仅存服务端，前端通过 Server Actions 调用
- **PII 处理：** V1 不收集真实姓名、电话等敏感信息
- **数据删除：** 用户删除账号时级联删除所有衣物图片与记录
- **GDPR / 隐私合规：** 提供数据导出与永久删除功能

## 8. 设计与体验 Design & UX Requirements

### 8.1 设计原则

1. **极简优雅**（与时尚定位匹配）：大量留白、克制配色、高级感字体
2. **AI 隐于幕后**（让结果说话）：不强调"AI 在工作"，强调"穿搭给你"
3. **手机优先**（90% 使用场景是早晨手机操作）：单手可操作、大点击区
4. **审美自洽**（产品本身就要有审美）：用产品 UI 证明我们懂搭配

### 8.2 视觉风格定调

- **主色：** 米白 #FAF8F5 / 墨黑 #1A1A1A / 焦糖点缀 #C8956D
- **字体：** PingFang SC（中文）/ Inter（英文）
- **图标：** 线性极简（Heroicons / Lucide）
- **圆角：** 12px（卡片）/ 999px（按钮）
- **阴影：** 极轻（shadow-sm）
- **动效：** 克制（淡入淡出 < 300ms，不要弹跳）

### 8.3 核心用户流程

**主流程 A：首次建库**
```
1. 落地页 → 点击"开始建立我的衣橱"
2. 注册 / 登录
3. 引导页 → "拍下你的衣服，让 AI 来认识它们"
4. 上传页 → 选择多张照片或拍照
5. Loading（5-30 秒）→ 显示进度 + 趣味文案
6. 识别结果页 → 网格展示，可逐项编辑
7. 完成 → 引导到首页
```

**主流程 B：日常推荐**
```
1. 首页（衣橱网格 + 醒目"今天穿什么"按钮）
2. 点击 → 场合输入页（5 个快捷标签 + 文字框）
3. 输入 → Loading 5-8 秒（伴随穿搭金句）
4. 结果页 → 2-3 套方案，滑动切换
5. 点击展开 → 详细解释
6. "今天就穿这套" → 记录入库
```

**主流程 C：付费转化**
```
1. 用户达到免费额度上限（建库 2 件 或 推荐 3 次）
2. 触发付费墙 → 价值展示 + 订阅选项
3. 选择套餐 → 支付
4. 支付成功 → 解锁全部功能
```

### 8.4 关键页面

- **P1：** 落地页 / 首页（含衣橱预览）
- **P2：** 注册 / 登录页
- **P3：** 衣物上传页
- **P4：** 衣橱网格视图
- **P5：** 场合输入页
- **P6：** 搭配方案展示页（核心 Wow Moment）
- **P7：** 搭配方案详情页
- **P8：** 付费墙
- **P9：** 订阅管理页

### 8.5 关键交互细节

- **Loading 体验：** 不要转圈圈，用文案传递"AI 正在审美"的体感
  - 建库中："正在认识你的衣服 1/10 件..."
  - 推荐中："正在调用审美库..." → "正在搭配..." → "正在写理由..."
- **搭配方案展示：** 单品图片用「拼贴」而非「展示」，营造杂志感
- **解释文案：** "因为今天面试需要干练感，米白衬衫降低正式度的同时增加亲和力" >>> "这套穿搭很合适"
- **空状态：** 衣橱为空时，"先拍几件衣服给我看看吧"
- **付费墙文案：** 强调价值而非限制（"解锁无限搭配，每天 30 秒搞定穿搭"）

### 8.6 可访问性

- 颜色对比度 ≥ 4.5:1
- 所有交互元素 ≥ 44pt 点击区
- 图片有 alt text
- 关键 CTA 按钮使用语义化标签

| 季度订阅 | 200 | ¥16.3 | ¥3,260 |
| 年度订阅 | 175 | ¥8.25 | ¥1,444 |
| **合计** | 500（10% 转化） | - | **¥7,079（≈ $985）** |

### 12.5 收支平衡分析

- 5,000 MAU 月成本 $1,020 vs 月收入 $985 → 接近盈亏平衡
- 10,000 MAU 起进入正向现金流
- 关键杠杆：提高付费转化率（5% → 8% 即可显著改善）

### 12.6 成本优化策略

1. **分级模型：** 免费用户用 Haiku，付费用户用 Sonnet
2. **建库优化：** 用 Gemini 2.0 Flash 做建库（成本降 30x，质量基本可接受）
3. **缓存激进：** System Prompt 命中率 > 95%
4. **图片压缩：** 上传前 resize 到 1024px
